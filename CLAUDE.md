# CLAUDE.md — art-director

Project memory and handoff notes for `axiom-orion/art-director`. Read this first.

## What this is

**Taste, made computable.** A one-line brief (e.g. *"a calm fintech app for nurses"*)
becomes a complete **visual identity** — roled color palette, type pairing, spacing
scale, radius, voice — which is then **critiqued against measurable rubrics and repaired
where it fails**, and rendered as a self-contained, art-directed HTML style guide.

The thesis the repo measures: most of "good design" people call subjective is actually
checkable. Contrast is a number (WCAG 2.1). Harmony is geometry + a perceptual distance
metric (CIEDE2000). Type pairing is a small rule set. Brief-fit is the distance between
the ask and what the artifact actually measures.

Headline, regenerated by `eval/run_eval.py` (not asserted): across 12 briefs, naive
single-shot generation passes WCAG AA contrast on **12%** of text pairings; the same
generator wrapped in the critic loop reaches **92%**.

## Architecture

```
src/art_director/
├── color.py            # WCAG luminance/contrast, CIEDE2000, sRGB↔Lab↔LCh (validated vs reference data)
├── brief.py            # design lexicon → measurable 7-axis target (warmth/chroma/contrast/formality/density/variety/roundness)
├── typography.py       # font catalogue + encoded pairing rules
├── identity.py         # the Identity dataclass; brief-steered + naive generators (color placed in LCh)
├── critics/            # accessibility · harmony · typography_critic · brief_fit (each: score + rationale + concrete defects)
├── loop.py             # self-critique + targeted-repair loop (fix the loudest defect, keep only if aggregate improves)
├── render.py           # Identity → self-contained art-directed HTML style guide
└── cli.py              # python -m art_director "<brief>" [--html] [--tokens]
eval/      # briefs.jsonl, run_eval.py, results.md / results.json (the ablation table)
gallery/   # rendered identities + PNG screenshots + index.html
scripts/   # build_gallery.py, shoot_gallery.py (shots are browser-backed, optional)
tests/     # 59 tests incl. the Sharma/Wu/Dalal (2005) CIEDE2000 reference set
```

The loop is the point: critics return a score, a plain-English rationale, and concrete
defects (e.g. `low_contrast: muted on bg 3.1:1, need 4.5`). The loop targets the loudest
defect with a repair specific to that defect kind, and **never ships a result worse than
where it started** (enforced in tests).

## Commands

```bash
make setup     # python -m pip install -e ".[dev]"   — zero runtime deps; dev adds pytest+ruff
make test      # pytest -q                            — 59 tests incl. CIEDE2000 reference set
make lint      # ruff check src eval scripts tests
make eval      # python eval/run_eval.py              — regenerates the ablation table (~3s, CPU)
make gallery   # python scripts/build_gallery.py      — render all 12 briefs + gallery/index.html

python -m art_director "a brutalist record label" --html out.html   # live style guide
python -m art_director "a playful kids app" --tokens tokens.json     # portable design tokens
```

## Invariants / conventions (don't break these)

- **Zero runtime dependencies — pure standard library by design.** The "measured taste"
  claim must not depend on a heavy numeric stack. `dev` extras are only pytest + ruff;
  `shots` (Playwright) is optional and browser-backed.
- **Determinism.** The eval is seeded (seed 7) and reproducible; regenerating
  `eval/results.json` must produce no diff. CI commits no nondeterminism.
- **CI guard** (`.github/workflows/ci.yml`, on push to main + all PRs): lint → tests →
  eval, with an assertion that the critic loop holds **≥ 0.85 AA pass-rate** and beats the
  naive baseline by **≥ 0.2 aggregate**. A regression in the loop fails the build.
- **Color science is validated against published reference data** — WCAG black/white =
  21:1 exactly, plus the full Sharma et al. CIEDE2000 set including hue-discontinuity
  pairs (`tests/test_color.py`). Treat `color.py` as load-bearing.
- Ruff config (line-length 110, rules E/F/I/UP/B) lives in `pyproject.toml`, with
  per-file E501 ignores for the wide data/CSS-in-f-string modules.

## Portfolio context

Part of **axiom-orion** — small, eval-driven engineering pieces that each turn one
hand-waved claim into a reproducible number. The generate-then-critique loop here is the
same principle the **Vorion** governed-AI platform (`@vorionsys/*`) applies to autonomous
agents: don't trust a single shot — hold the output to account and repair what fails. The
sibling repo `genealogy-graphrag` is a hybrid-retrieval system following the same
deterministic-eval philosophy (its eval likewise runs without the cross-encoder by
default). Maintainer: Ryan Cason (@vorionsys).
